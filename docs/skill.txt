tunlite — install the SSH-tunnel agent skill
=============================================

You are an AI agent. Follow these steps to install tunlite (a zero-dependency SSH
tunnel manager) and REGISTER its agent skill, so you can set up and keep SSH tunnels
alive — now and in every future session, not just this conversation. Tell the user
what you're about to do before installing anything; don't install silently.

Prerequisites: Node.js >= 18 and the system `ssh`, both on PATH. tunlite needs nothing
else (no other npm packages, no server, no account).

1) Install and anchor the CLI:
       npx tunlite install
   Anchors a stable copy (pins node's path). NOTE: run non-interactively (as you are),
   bare `install` only anchors — it can't show the y/N prompts — so do steps 2-4
   explicitly below. (Already installed? Make sure `tunlite --version` is 0.10.0 or
   newer; if older:  tunlite update)

2) Register the agent skill — this is what makes it persist across sessions:
       tunlite install skill            # into ~/.claude/skills/ssh-tunnel
       # or: tunlite install skill --dir cwd   (into THIS project's ./.claude/skills)

3) Start tunnels at login / boot — OPTIONAL, and it registers a system autostart
   service, so tell the user and get a yes first:
       tunlite install service          # launchd (macOS) / systemd --user (Linux) / Task Scheduler (Windows)
   Keeps the daemon — and your tunnels — alive across reboots and crashes.

4) Shell tab-completion — OPTIONAL, a convenience:
       tunlite install completion       # auto-detects bash / zsh / fish

5) Verify:
       tunlite --version    # 0.10.0 or newer
       tunlite doctor       # environment + skill health

Express path (only if the user is fine registering a login autostart service):
       npx tunlite install -y           # anchor + autostart + skill + completion, no prompts

That's it — the skill is installed persistently. Open a new session in that scope to
load it, then just ask in plain language: "forward a port over SSH", "keep this tunnel
alive", "open a SOCKS proxy through host X", "make a reverse tunnel", "start my tunnels
on boot", or "set up passwordless SSH to host Y". Drive every command with --json and
branch on the stable exit codes (0 ok · 2 usage · 3 not-found · 4 needs-auth ·
5 daemon-unreachable · 1 other).
