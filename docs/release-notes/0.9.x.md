## v0.9.5 — 2026-06-18

Critical fix: `tunlite update` no longer fails once the repo archive passes ~1 MB. The
fetcher buffered the whole tarball into spawnSync's 1 MB default and was killed with a
misleading `need curl or wget`; it now streams to a file (any size) and reports the
real download error.

Full notes → [CHANGELOG.md](https://github.com/yuanyuanzijin/tunlite/blob/master/CHANGELOG.md)

严重修复:仓库 archive 超过约 1 MB 后,`tunlite update` 不再失败。此前取包把整个 tarball
读进了 spawnSync 默认 1 MB 缓冲区、被杀掉还误报 `need curl or wget`;现在改为下载到文件
(多大都行),并报出真实的下载错误。

完整说明 → [CHANGELOG.md](https://github.com/yuanyuanzijin/tunlite/blob/master/CHANGELOG.md)

## v0.9.4 — 2026-06-18

Robustness & agent-contract fixes — no new features. Highlights: `add` no longer
silently overwrites an existing tunnel (was data loss); install/update survive a
symlinked install dir (Fedora Silverblue/CoreOS, macOS `/tmp`); `<command> --help`
works for every command; `logs`/`status`/`list` return the documented not-found (`3`)
for unknown tunnels and empty `--tag` selections.

Full notes → [CHANGELOG.md](https://github.com/yuanyuanzijin/tunlite/blob/master/CHANGELOG.md)

> ⚠️ **On 0.9.4 or earlier? `tunlite update` is broken** (it choked on the repo
> archive once it passed 1 MB) and can't pull its own fix — reinstall once:
> `npx tunlite@latest install` (0.9.5+ fixes it for good).

机制健壮性与 agent 契约修复,无新功能。要点:`add` 不再静默覆盖已有隧道(此前丢数据);
安装目录经符号链接时安装/更新不再失效(Fedora Silverblue/CoreOS、macOS `/tmp`);
`<命令> --help` 对每个命令都可用;`logs`/`status`/`list` 对不存在的隧道和空 `--tag`
返回文档承诺的 not-found(`3`)。

完整说明 → [CHANGELOG.md](https://github.com/yuanyuanzijin/tunlite/blob/master/CHANGELOG.md)

> ⚠️ **在 0.9.4 或更早?`tunlite update` 是坏的**(仓库 archive 超过 1 MB 后取包失败),
> 且救不了自己 —— 重装一次:`npx tunlite@latest install`(0.9.5 起彻底修好)。

## v0.9.0 — 2026-06-16

First public release. Named, declarative SSH tunnels supervised by a zero-dependency
daemon (auto-reconnect, OS autostart, passwordless setup), `--json` + stable exit codes
for agents. Highlights: ProxyJump/bastion (`--jump`), multiple forwards per tunnel,
`set` to edit in place, tags/groups, plus reconnect-backoff, anti-double-daemon, and a
batch of security hardening (ssh-arg-injection closed, owner-locked control socket,
redacted webhook URLs, https/file-only self-update).

Full notes → [CHANGELOG.md](https://github.com/yuanyuanzijin/tunlite/blob/master/CHANGELOG.md)

> ⚠️ `tunlite update` is broken on this version (archive >1 MB) — reinstall once:
> `npx tunlite@latest install` (0.9.5+ fixes it).

首个公开版本。零依赖守护进程托管的命名式声明式 SSH 隧道(自动重连、开机自启、免密打通),
每个命令 `--json` + 稳定退出码、对 agent 友好。要点:跳板机(`--jump`)、单隧道多转发、
`set` 就地编辑、标签分组,以及重连退避、防双守护进程、一批安全加固(关闭 ssh 参数注入、
属主锁定控制 socket、webhook URL 脱敏、自更新仅 https/file)。

完整说明 → [CHANGELOG.md](https://github.com/yuanyuanzijin/tunlite/blob/master/CHANGELOG.md)

> ⚠️ 此版本 `tunlite update` 是坏的(archive >1 MB)—— 重装一次:
> `npx tunlite@latest install`(0.9.5 起修好)。
