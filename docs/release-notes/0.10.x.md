## v0.10.0 — 2026-06-19

A breaking release. Forwards are now ssh-native, repeatable `-L`/`-R`/`-D` (the old
`add local|remote|dynamic` and the separate `forward` command are gone; `set` is the
sole forward editor); the control verbs `up`/`down` become `enable`/`disable` and now
require a target — a name, `--tag`, or the literal `all` — with did-you-mean for retired
or mistyped verbs; `install` is one guided command (the `--service`/`--skill`/`--completion`
flags are gone — use `install service|skill|completion`); and `export` prints only
`{ version, tunnels }`. Also new: a daemon-less foreground `tunlite run` for
container/systemd entrypoints, and an endpoint-conflict lock + `blocked` state. Your
existing tunnel config keeps working — only the commands changed.

Upgrade: `tunlite update` (or `npx tunlite@latest install`); confirm with `tunlite --version`.
